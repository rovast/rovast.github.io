Skywalking PHP 内核拓展增加自定义 Tag

2021-03-20 0 0 评论 字数统计: 1.2k(字) 阅读时长: 5(分)

一、需求背景和目的

1.1、需求

业务方希望能够根据中台的属性——Channel 作为查询条件。已知目前在业务系统中，会通过在 HTTP Header 中标记的方式来传递对应信息，目前传递的 Header 信息为：

其中，默认我们在代码中传递的是 x-channel，但是经过 Nginx 之后，会被修改为 HTTP_X_CHANNEL。

1.2、目的

针对上述需求，通过调研 Skywalking Query Protocol 后，我们知道，可以将 Tag 属性作为 Trace 信息的查询条件[1]。并且根据 OpenTracing 的标准[2]，用户自定义的，业务相关的内容，添加到 Span 的 Tag 里，是一种较为科学的处理方式。

2.2 修改思路

图中高亮的区域就是需要修改的环节。好在源码中每次的 Tag 初始化写法都一样，如下：

所以我们可以很轻易的定位到影响的范围，其实就是上面图中标注的区域。然后我们去 http header 里读取需要处理的 header key 值即可。核心处理思路是：在 Tag 初始化环节注入 channel 信息。处理流程如下：

  1. 定位到 tag 初始化环节
  2. 读取 header 里的 channel 信息（我们同时处理下 appid 和 appkey 信息）
  3. 把 channel 信息注入到 tag 里

三、代码实现说明

3.1 关键处理函数

  • zend_hash_str_find

    根据 string 类型的键值，在数组(hash)中，查找对应的元素。其返回是 zval

  • add_assoc_string

    即：add associative string to array，向数组中添加 string 类型的元素。机智如你，肯定还有 add_assoc_long 等系列函数

  • EG(symbol_table)

    即：executor_globals.symbol_table，就是获取执行阶段中的全局变量。我们可以获取到全局的 $_SERVER，进而读取指定的 header。

  • Z_STRVAL_P

    即：根据 pointer 获取到 str 类型的 zval 值。等于 Z_STRVAL(*(zval_p))，进而 ZSTR_VAL(Z_STR(zval))
